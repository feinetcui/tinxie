# 亲子听写对战 H5 应用 - 开发计划

## 项目概述
一个支持家长和孩子进行听写比赛的H5应用，控制端拍照识别词库，选手端手写汉字作答。

## 技术架构

```
┌─────────────────────────────────────────────────┐
│                    前端 (H5)                      │
│  ┌──────────────┐    ┌──────────────────────┐   │
│  │   控制端      │    │      选手端           │   │
│  │  (家长/老师)  │    │    (学生)             │   │
│  └──────┬───────┘    └──────────┬───────────┘   │
│         │                       │                │
│         └───────────┬───────────┘                │
│                     │ WebSocket                  │
├─────────────────────┼───────────────────────────┤
│              后端 Node.js                        │
│  ┌─────────────────┴───────────────────────┐   │
│  │  房间管理 │ 实时通信 │ 多模态AI(OCR+手写) │   │
│  └─────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

## 技术选型

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 前端框架 | 原生 HTML5 + CSS3 + JS | 轻量，无构建工具依赖 |
| 实时通信 | WebSocket (ws库) | 房间同步、状态推送 |
| 二维码 | qrcode.js | 生成房间二维码 |
| 拼音显示 | pinyin-pro | 汉字转拼音 |

| 后端 | Node.js + Express | 轻量服务端 |
| 响应式 | CSS Media Query + Flex/Grid | 适配移动端和PC |

## 核心功能流程

### 控制端（家长/老师）

```
1. 打开 host.html → 创建房间 → 生成房间二维码
2. 拍照课本页面 → 调用OCR识别文字
3. 从识别结果中勾选要听写的词语
4. 设置计时时长（5/10/15/20秒或自定义）
5. 点击"开始听写" → 选手端收到指令
6. 实时查看选手书写状态和结果
7. 所有人完成后 → 显示成绩单（所有词语 + 正误）
8. 可选择：进入错题练习模式 / 结束
```

### 选手端（学生）

```
1. 扫描二维码 → 输入昵称加入房间
2. 等待控制端开始
3. 收到听写指令 → 显示拼音 + 倒计时（10秒）
4. 在田字格内手写汉字 → 提交（或倒计时结束自动收卷）
5. 显示本题结果（正确/错误 + 正确答案）
6. 等待下一题
7. 全部完成 → 显示所有词语成绩单
8. 进入错题练习模式 → 每个错题练习3次
9. 练习完成 → 显示最终成绩
```

## 关键实现细节

### 1. 房间管理
- 房间ID: 4位数字（如 8832）
- WebSocket 消息协议:
  ```json
  // 创建房间
  { "type": "create_room", "roomId": "8832" }
  
  // 加入房间
  { "type": "join_room", "roomId": "8832", "nickname": "小明" }
  
  // 开始听写（含倒计时设置）
  { "type": "start_round", "words": ["春天", "花朵", "蝴蝶"], "timeLimit": 10 }
  
  // 控制端修改计时设置（听写过程中可调整）
  { "type": "update_time_limit", "timeLimit": 15 }
  
  // 倒计时结束
  { "type": "time_up", "word": "春天" }
  
  // 选手提交答案
  { "type": "submit_answer", "word": "春天", "image": "base64..." }
  
  // 广播结果
  { "type": "answer_result", "player": "小明", "word": "春天", "correct": true, "recognized": "春天" }
  
  // 听写完成，发送成绩单
  { "type": "dictation_complete", "results": [
    {"word": "春天", "correct": true}, 
    {"word": "燕子", "correct": false}
  ]}
  
  // 开始错题练习
  { "type": "start_practice", "words": ["燕子", "溪流", "栽树"], "round": 1, "totalRounds": 3 }
  
  // 练习提交结果
  { "type": "practice_result", "word": "燕子", "round": 1, "correct": true }
  
  // 最终成绩
  { "type": "final_score", "scores": [{ "player": "小明", "correct": 8, "total": 10, "practiceCorrect": 2, "practiceTotal": 3 }] }
  ```

### 2. AI 识别流程
- 控制端拍照 → base64 图片 → 发送到后端
- 后端调用Agnes AI（免费，OpenAI兼容格式）→ 返回识别的文字列表
- 前端展示识别结果 → 用户勾选词语 → 生成听写列表

### 3. 手写识别流程
- 选手端 Canvas 手写 → 生成图片 base64
- 提交到后端 → AI判断手写是否正确
- 返回判定结果 → 与正确答案比对

### 4. 拼音 + 田字格显示
- 使用 pinyin-pro 库将汉字转拼音
- **上方**: 显示拼音（大字体，清晰可读）
- **下方**: Canvas绘制田字格，选手在格子内手写
- 格子数量 = 词语字数（如"春天"2个格子）

### 5. 响应式设计
```css
/* 移动端 */
@media (max-width: 768px) {
  .canvas-container { width: 100%; height: 60vh; }
}

/* PC端 */
@media (min-width: 769px) {
  .canvas-container { width: 600px; height: 400px; }
}
```

## 听写规则
- **限时模式**: 每个词固定倒计时（默认10秒）

## 常见问题

### Q1: 部署后 WebSocket 连接失败？
A: 检查 `common.js` 中的 WebSocket 地址是否正确。Zeabur 会自动处理 HTTPS，确保使用 `wss://` 协议。

### Q2: 拍照功能不工作？
A: 浏览器要求 HTTPS 才能访问相机。Zeabur 默认提供 HTTPS，确保通过 HTTPS 访问。

### Q3: AI 识别返回错误？
A: 检查环境变量 `AGNES_API_KEY` 是否正确配置。

### Q4: 如何更新部署？
A: 在本地修改代码后，执行：
```bash
git add .
git commit -m "更新说明"
git push
```
Zeabur 会自动重新部署。
