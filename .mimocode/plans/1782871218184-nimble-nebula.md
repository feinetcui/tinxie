# 云平台对比分析：Wispbyte、Railway、Hugging Face Spaces

## 项目需求回顾
亲子听写对战应用需要：
- **WebSocket 实时通信** - 房间同步、状态推送
- **Node.js 后端** - Express + ws 库
- **AI API 调用** - OCR 和手写识别（Agnes AI）
- **静态资源托管** - HTML/CSS/JS 前端

---

## 1. Railway

### 特点
| 项目 | 说明 |
|------|------|
| **类型** | 容器化部署平台（类似 Heroku） |
| **WebSocket** | ✅ 完全支持 |
| **Node.js** | ✅ 原生支持（Nixpacks 自动检测） |
| **定价** | 免费版 $0/月（$1 额度），Hobby $5/月（含 $5 额度） |
| **免费额度** | 0.5 GB RAM, 1 vCPU |
| **GitHub 集成** | ✅ 推送自动部署 |
| **自定义域名** | ✅ 支持，自动 TLS |

### 优点
- 完整的 Node.js 支持，无需改造代码
- WebSocket 原生支持
- 免费版可用（适合低流量应用）
- GitHub 推送自动部署
- 内置数据库支持（Postgres、Redis 等）

### 缺点
- 免费额度有限（$1/月）
- 超出免费额度需付费

### 适合度：⭐⭐⭐⭐⭐（非常适合）

## 推荐方案

### 首选：Railway
**理由：**
1. **国内可访问** - 已测试从国内可以正常访问 railway.com
2. **无需代码改造** - 当前 Node.js 代码可以直接部署
3. **WebSocket 原生支持** - 不需要 Durable Objects
4. **免费版可用** - 家庭使用流量低，$1 额度足够
5. **部署简单** - `railway up -y` 一键部署
6. **GitHub 集成** - 推送自动部署

### 部署步骤（Railway）
1. 安装 Railway CLI：`npm i -g @railway/cli`
2. 登录：`railway login`
3. 部署：`railway up -y`
4. 添加域名：`railway domain`
5. 设置环境变量：`AGNES_API_KEY`

如果不想切换平台，可以继续调试 Durable Objects 的 WebSocket 问题。
